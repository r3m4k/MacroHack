## 1. Общая архитектура

Pipeline решает задачу прогнозирования 9 процентных ставок рублёвой кривой доходности на 6 месяцев вперёд (октябрь 2025 — март 2026). Строятся две модели:

- **M1** — использует только данные кривой доходности и открытые данные. Не имеет доступа к поверхности implied volatility (IV).
- **M2** — расширяет M1 добавлением признаков, извлечённых из поверхности IV. Новые источники данных запрещены; допускается только перенастройка гиперпараметров.

Обе модели реализованы как per-tenor Ridge Regression: для каждого из 9 теноров обучается своя отдельная модель. Прогноз — рекурсивный (прогноз предыдущего месяца подставляется во вход для прогноза следующего).

Стек: `numpy`, `pandas`, `sklearn` (Ridge, StandardScaler, PCA, TimeSeriesSplit). Опционально — `statsmodels` (VAR-модель).

## 2. Блок 0. Конфигурация

```python
YIELD_TENORS = ["O/N", "1W", "2W", "1M", "2M", "3M", "6M", "1Y", "2Y"]
IV_TENORS = ["1M", "2M", "3M", "6M", "9M", "1Y", "2Y", "3Y", "4Y",
             "5Y", "6Y", "7Y", "8Y", "9Y", "10Y"]
TRAIN_START = "2019-03"
TRAIN_END = "2025-09"
TEST_START = "2025-10"
TEST_END = "2026-03"
WEIGHT_ON = 0.4
WEIGHT_OTHER = 0.6 / (len(YIELD_TENORS) - 1)  # = 0.075
```

**Что здесь происходит:**

- `YIELD_TENORS` — 9 теноров целевой кривой доходности. Это ровно те 9 столбцов, которые мы прогнозируем. O/N — RUONIA (overnight), остальные — ROISfix (свопы на RUONIA).
- `IV_TENORS` — 15 теноров поверхности вменённой волатильности. Используются только как источник дополнительных признаков в M2.
- Временные границы: обучающая выборка — 78 месяцев (март 2019 — сентябрь 2025), тестовая — 6 месяцев (октябрь 2025 — март 2026).
- Веса метрики: O/N получает вес 0.4 (наибольший), каждый из остальных 8 теноров — по 0.075 (в сумме 0.6). Это означает, что ошибка в прогнозе O/N карается в 5.3 раза сильнее, чем ошибка в любом другом теноре.

## 3. Блок 1. Загрузка данных

```python
def load_yield_curve(filepath: str) -> pd.DataFrame:
    dates = pd.date_range(TRAIN_START, TRAIN_END, freq="MS")
    np.random.seed(42)
    base = np.linspace(7.0, 9.0, len(YIELD_TENORS))
    data = np.tile(base, (len(dates), 1)) + np.random.randn(...) * 0.5
    df = pd.DataFrame(data, index=dates, columns=YIELD_TENORS)
    return df
```

**Что здесь происходит:**

- Функции `load_yield_curve` и `load_iv_surface` — точки входа данных в pipeline.
- Сейчас внутри **заглушки** с синтетическими данными для тестирования. Синтетическая кривая: `np.linspace(7.0, 9.0, 9)` создаёт нормальную восходящую кривую от 7% (O/N) до 9% (2Y), к которой добавляется случайный шум `± 0.5`.
- Для работы с реальными данными: раскомментировать `pd.read_csv(filepath, index_col=0, parse_dates=True)` и подставить путь к файлу.
- Ожидаемый формат: индекс = дата (ежемесячно), столбцы = названия теноров.

**Выход:** DataFrame размером (78, 9) для кривой доходности и (78, 15) для IV.

## 4. Блок 2. Feature Engineering — кривая доходности (M1)

Центральная часть pipeline. Класс `FeatureEngineer` принимает на вход сырую матрицу ставок (78 строк × 9 столбцов) и создаёт расширенную матрицу признаков (~79 столбцов). Каждая строка по-прежнему соответствует одному месяцу, но теперь вместо 9 «сырых» ставок модель получает ~79 вычисленных характеристик.

### 2.1. Лагированные значения

```python
for lag in [1, 2, 3]:
    lagged = df.shift(lag)
    lagged.columns = [f"{col}_lag{lag}" for col in df.columns]
    features = pd.concat([features, lagged], axis=1)
```

**Что делает код:** `df.shift(lag)` сдвигает весь DataFrame на `lag` строк вниз. Для строки «сентябрь 2025» `shift(1)` подставит значения из «августа 2025», `shift(2)` — из «июля», `shift(3)` — из «июня».

**Какие столбцы создаются (27 штук):**

| Столбец | Значение |
|---|---|
| `O/N_lag1` | Ставка O/N за 1 месяц до текущего |
| `O/N_lag2` | Ставка O/N за 2 месяца до текущего |
| `O/N_lag3` | Ставка O/N за 3 месяца до текущего |
| `1W_lag1` | Ставка 1W за 1 месяц до текущего |
| ... | ... (аналогично для всех 9 теноров) |
| `2Y_lag3` | Ставка 2Y за 3 месяца до текущего |

**Зачем:** Процентные ставки обладают сильной автокорреляцией — значение в прошлом месяце является лучшим предиктором значения в текущем. Лаги 2 и 3 добавляют модели «память» — она видит не точку, а траекторию.

**Побочный эффект:** Первые 3 строки DataFrame получают NaN (для марта 2019 нет данных за декабрь 2018). Эти строки позже удаляются через `dropna()`.

**Пример (для O/N):**

```
Дата         O/N    O/N_lag1  O/N_lag2  O/N_lag3
2025-06      15.0   NaN       NaN       NaN        ← (если это первая строка)
2025-07      15.5   15.0      NaN       NaN
2025-08      15.8   15.5      15.0      NaN
2025-09      16.0   15.8      15.5      15.0       ← все лаги заполнены
```

### 2.2. Спреды между тенорами

```python
features["spread_2Y_ON"] = df["2Y"] - df["O/N"]
features["spread_1Y_ON"] = df["1Y"] - df["O/N"]
features["spread_2Y_1Y"] = df["2Y"] - df["1Y"]
features["spread_6M_3M"] = df["6M"] - df["3M"]
features["spread_3M_1M"] = df["3M"] - df["1M"]

for lag in [1, 2]:
    for col in ["spread_2Y_ON", "spread_1Y_ON", "spread_2Y_1Y"]:
        features[f"{col}_lag{lag}"] = features[col].shift(lag)
```

**Что делает код:** Вычисляет разницу ставок между парами теноров в одном и том же месяце, а затем создаёт лагированные версии ключевых спредов.

**Какие столбцы создаются (11 штук):**

| Столбец | Формула | Экономический смысл |
|---|---|---|
| `spread_2Y_ON` | 2Y − O/N | Полный наклон кривой. Показывает, насколько рынок ожидает изменения ставки ЦБ на горизонте 2 лет. Отрицательный = инверсия (рынок ждёт снижения). |
| `spread_1Y_ON` | 1Y − O/N | Наклон на горизонте 1 года. Аналогично, но на более коротком горизонте. |
| `spread_2Y_1Y` | 2Y − 1Y | Наклон длинного конца. Показывает, ожидает ли рынок дальнейших изменений после первого года. |
| `spread_6M_3M` | 6M − 3M | Наклон середины кривой. Ожидания на ближайшие полгода. |
| `spread_3M_1M` | 3M − 1M | Наклон короткого конца. Ближайшие ожидания (1–3 месяца). |
| `spread_2Y_ON_lag1` | Спред 2Y−O/N за 1 месяц назад | Как менялся наклон — направление изменения ожиданий. |
| `spread_2Y_ON_lag2` | Спред 2Y−O/N за 2 месяца назад | Более долгосрочная динамика наклона. |
| `spread_1Y_ON_lag1` | Аналогично | — |
| `spread_1Y_ON_lag2` | Аналогично | — |
| `spread_2Y_1Y_lag1` | Аналогично | — |
| `spread_2Y_1Y_lag2` | Аналогично | — |

**Зачем:** Спреды кодируют форму кривой — информацию, которую невозможно извлечь из отдельных уровней. Если O/N = 16% и 2Y = 10%, спред = −6%. Если O/N = 8% и 2Y = 2%, спред тоже = −6%. Форма одинаковая, хотя уровни разные. Лагированные спреды показывают, как менялась форма кривой, что критично для предсказания: если инверсия углубляется месяц за месяцем, рынок всё сильнее закладывает снижение ставки.

### 2.3. Скользящие средние

```python
for window in [3, 6]:
    rolled = df.rolling(window).mean()
    rolled.columns = [f"{col}_ma{window}" for col in df.columns]
    features = pd.concat([features, rolled], axis=1)
```

**Что делает код:** `df.rolling(window).mean()` считает среднее значение за последние `window` месяцев для каждого тенора. Для окна 3: среднее за текущий + 2 предыдущих месяца. Для окна 6: среднее за текущий + 5 предыдущих.

**Какие столбцы создаются (18 штук):**

| Столбец | Формула |
|---|---|
| `O/N_ma3` | mean(O/N за последние 3 месяца) |
| `O/N_ma6` | mean(O/N за последние 6 месяцев) |
| `1W_ma3` | mean(1W за последние 3 месяца) |
| ... | ... (аналогично для всех 9 теноров × 2 окна) |

**Зачем:** Скользящие средние выполняют две функции. Во-первых, сглаживание шума — если ставка в один месяц скакнула из-за единичного события (например, налоговый период), MA3/MA6 отфильтруют этот выброс и покажут «истинный» тренд. Во-вторых, неявная генерация сигнала «отклонение от тренда»: если текущее значение O/N = 16%, а MA6 = 14%, модель видит, что ставка на 2% выше своего среднего за полгода. Это сигнал, что ставка может вернуться к среднему (mean reversion) или что тренд усиливается.

**Побочный эффект:** MA6 создаёт NaN в первых 5 строках.

### 2.4. Дельты (месячные изменения)

```python
delta1 = df.diff(1)
delta1.columns = [f"{col}_delta1" for col in df.columns]
features = pd.concat([features, delta1], axis=1)

delta3 = df.diff(3)
delta3.columns = [f"{col}_delta3" for col in df.columns]
features = pd.concat([features, delta3], axis=1)
```

**Что делает код:** `df.diff(n)` вычисляет разницу между текущим значением и значением `n` строк назад. `diff(1)` — месячное изменение. `diff(3)` — квартальное изменение (разница с 3 месяцами назад).

**Какие столбцы создаются (18 штук):**

| Столбец | Формула | Смысл |
|---|---|---|
| `O/N_delta1` | O/N(t) − O/N(t−1) | Месячное изменение O/N в б.п. |
| `O/N_delta3` | O/N(t) − O/N(t−3) | Квартальное изменение O/N |
| `1W_delta1` | 1W(t) − 1W(t−1) | Месячное изменение 1W |
| ... | ... | ... |

**Зачем:** Дельты ловят два эффекта.

1. **Momentum (инерция):** Если O/N растёт 3 месяца подряд (все delta1 > 0), вероятно, продолжит расти. Модель учится распознавать устойчивые тренды.
2. **Скорость изменения:** delta1 = +0.1% — медленный рост. delta1 = +2.0% — резкое повышение ключевой ставки ЦБ. Модель реагирует на скорость по-разному.

delta3 добавляет более устойчивый сигнал: месячные delta1 могут колебаться, но если delta3 > 0, рост подтверждён на квартальном горизонте.

**Пример:**

```
Дата      O/N     O/N_delta1   O/N_delta3
2025-06   15.0    +0.3         +0.8
2025-07   15.5    +0.5         +1.0
2025-08   15.8    +0.3         +1.1
2025-09   16.0    +0.2         +1.0    ← рост замедляется (delta1 падает),
                                         но квартальный тренд стабилен
```

### 2.5. Кривизна (curvature)

```python
features["curvature_1"] = 2 * df["1M"] - df["O/N"] - df["6M"]
features["curvature_2"] = 2 * df["1Y"] - df["3M"] - df["2Y"]
```

**Что делает код:** Вычисляет кривизну кривой доходности на двух участках. Формула `2 × mid − short − long` — это дискретный аналог второй производной. Если кривая линейна между точками short и long, то `mid = (short + long) / 2`, и кривизна = 0.

**Какие столбцы создаются (2 штуки):**

| Столбец | Формула | Какой участок кривой |
|---|---|---|
| `curvature_1` | 2×1M − O/N − 6M | Короткий участок (O/N → 1M → 6M) |
| `curvature_2` | 2×1Y − 3M − 2Y | Длинный участок (3M → 1Y → 2Y) |

**Зачем:** Кривизна ловит нелинейность формы кривой, которую спреды не видят.

- **curvature > 0** — «горб» (ставка в середине выше, чем линейная интерполяция между краями). Рынок считает, что среднесрочные ставки будут выше, чем предполагает линейный тренд.
- **curvature < 0** — «впадина». Среднесрочные ставки ниже ожидаемого.
- **curvature ≈ 0** — кривая линейна на этом участке.

Резкое изменение кривизны — ранний индикатор смены режима. Например, перед циклом снижения ставки кривизна на длинном конце часто уходит в отрицательную зону.

### 2.6. PCA-компоненты кривой

```python
pca = PCA(n_components=3)
pca_vals = pca.fit_transform(df.values)
features["pca_level"] = pca_vals[:, 0]
features["pca_slope"] = pca_vals[:, 1]
features["pca_curvature"] = pca_vals[:, 2]
```

**Что делает код:** PCA (Principal Component Analysis) — метод понижения размерности. Он берёт 9 исходных столбцов (теноров) и находит 3 новых «оси», вдоль которых данные варьируются больше всего. Технически: находит собственные векторы ковариационной матрицы 9×9.

**Какие столбцы создаются (3 штуки):**

| Столбец | % дисперсии (типично) | Экономическая интерпретация |
|---|---|---|
| `pca_level` | ~80% | **Уровень** — все ставки двигаются вверх или вниз одновременно. Высокое значение = все ставки высоко (ужесточение ДКП). |
| `pca_slope` | ~10–15% | **Наклон** — короткие ставки идут в одну сторону, длинные — в другую. Положительный = нормальная кривая, отрицательный = инверсия. |
| `pca_curvature` | ~3–5% | **Кривизна** — середина кривой отклоняется от линии между краями. Аналогично `curvature_1/2`, но вычислен математически оптимально. |

**Зачем:** Три PCA-компоненты сжимают 9 коррелированных переменных в 3 некоррелированных. Это помогает Ridge-модели: вместо 9 связанных столбцов (ставки O/N и 1W двигаются почти синхронно) модель работает с 3 независимыми осями. Снижается мультиколлинеарность, повышается устойчивость коэффициентов.

В финансовой литературе первые 3 PCA-компоненты кривой доходности — это стандартная декомпозиция, известная с работ Litterman & Scheinkman (1991).

## 5. Блок 2 (продолжение). Feature Engineering — поверхность IV (M2)

Метод `iv_surface_features()` добавляет признаки из поверхности вменённой волатильности. Эти фичи используются **только в модели M2**.

**Критический принцип:** Все IV-фичи сдвинуты на 1 месяц через `.shift(1)`. Без этого мы использовали бы IV текущего месяца для прогноза текущего месяца — это утечка данных (look-ahead bias). Со сдвигом мы используем IV прошлого месяца, что корректно: мы знаем IV за сентябрь, когда прогнозируем октябрь.

### 3.1. Лагированные значения IV

```python
iv_lagged = iv.shift(1)
iv_lagged.columns = [f"iv_{col}_lag1" for col in iv.columns]
features = pd.concat([features, iv_lagged], axis=1)
```

**Что делает код:** Берёт значение IV каждого из 15 теноров за предыдущий месяц.

**Какие столбцы создаются (15 штук):**

| Столбец | Значение |
|---|---|
| `iv_1M_lag1` | IV тенора 1M за предыдущий месяц |
| `iv_2M_lag1` | IV тенора 2M за предыдущий месяц |
| ... | ... |
| `iv_10Y_lag1` | IV тенора 10Y за предыдущий месяц |

**Зачем:** Уровень IV — прямой индикатор рыночной неопределённости. Высокая IV на коротких тенорах (1M, 3M) = рынок ожидает скорых резких движений ставок (например, перед заседанием ЦБ). Высокая IV на длинных тенорах (5Y, 10Y) = долгосрочная структурная неопределённость (инфляция, геополитика).

---

### 3.2. Термструктура IV (спреды)

```python
features["iv_spread_10Y_1M"] = iv["10Y"].shift(1) - iv["1M"].shift(1)
features["iv_spread_5Y_1Y"] = iv["5Y"].shift(1) - iv["1Y"].shift(1)
features["iv_spread_2Y_6M"] = iv["2Y"].shift(1) - iv["6M"].shift(1)
```

**Какие столбцы создаются (3 штуки):**

| Столбец | Формула | Смысл |
|---|---|---|
| `iv_spread_10Y_1M` | IV(10Y) − IV(1M) | Наклон термструктуры волатильности. Если положительный — долгосрочная неопределённость выше краткосрочной (нормальная ситуация). Если отрицательный — рынок нервничает прямо сейчас больше, чем на горизонте (стрессовый период). |
| `iv_spread_5Y_1Y` | IV(5Y) − IV(1Y) | Спред на среднем участке. Помогает различить, локализован ли стресс на коротком конце или распространяется. |
| `iv_spread_2Y_6M` | IV(2Y) − IV(6M) | Спред ближнего участка. Чувствителен к ожиданиям конкретных действий ЦБ. |

**Зачем:** Термструктура IV несёт информацию, отсутствующую в уровнях. Два сценария: (а) IV(1M) = 20%, IV(10Y) = 30% — спред +10%, нормальный режим; (б) IV(1M) = 35%, IV(10Y) = 25% — спред −10%, рынок в стрессе, ожидает шок в ближайший месяц. В обоих случаях среднее IV одинаково, но спреды дают радикально разные сигналы.

---

### 3.3. Дельты IV

```python
iv_delta = iv.diff(1).shift(1)
iv_delta.columns = [f"iv_{col}_delta1" for col in iv.columns]
features = pd.concat([features, iv_delta], axis=1)
```

**Что делает код:** Сначала `diff(1)` вычисляет месячное изменение IV, затем `shift(1)` сдвигает на месяц назад (предотвращение look-ahead).

**Какие столбцы создаются (15 штук):**

| Столбец | Формула |
|---|---|
| `iv_1M_delta1` | IV_1M(t−1) − IV_1M(t−2) |
| `iv_2M_delta1` | IV_2M(t−1) − IV_2M(t−2) |
| ... | ... |

**Зачем:** Изменение IV — это сигнал. Если IV резко выросла, значит рынок начал перестраховываться. Эмпирически, рост IV на коротких тенорах за 1–2 месяца до заседания ЦБ предшествует неожиданным решениям (повышение ставки больше ожидаемого). Модель учится: «delta_iv_1M = +5% → в следующем месяце O/N подскочит».

---

### 3.4. Скользящие средние IV

```python
iv_ma3 = iv.rolling(3).mean().shift(1)
iv_ma3.columns = [f"iv_{col}_ma3" for col in iv.columns]
features = pd.concat([features, iv_ma3], axis=1)
```

**Какие столбцы создаются (15 штук):**

| Столбец | Формула |
|---|---|
| `iv_1M_ma3` | Среднее IV_1M за 3 месяца (сдвинутое на 1 месяц) |
| ... | ... |

**Зачем:** Сглаженная IV показывает «базовый» уровень рыночной нервозности. Разница между текущей IV и MA3 — это «всплеск» волатильности, который может быть временным (и тогда ставки вернутся) или устойчивым (и тогда ставки сдвинутся).

---

### 3.5. PCA-компоненты поверхности IV

```python
iv_clean = iv.dropna()
if len(iv_clean) > 5:
    pca_iv = PCA(n_components=3)
    pca_vals = pca_iv.fit_transform(iv_clean.values)
    pca_df = pd.DataFrame(pca_vals, index=iv_clean.index,
                          columns=["iv_pca_1", "iv_pca_2", "iv_pca_3"])
    pca_df = pca_df.shift(1)  # look-ahead prevention
    features = features.join(pca_df, how="left")
```

**Какие столбцы создаются (3 штуки):**

| Столбец | Интерпретация |
|---|---|
| `iv_pca_1` | Общий уровень волатильности по всей поверхности. Растёт — рынок в стрессе. |
| `iv_pca_2` | Наклон термструктуры IV. Короткая vs длинная волатильность. |
| `iv_pca_3` | Кривизна поверхности IV. Нелинейности в распределении неопределённости. |

**Зачем:** Аналогично PCA кривой доходности — сжимаем 15 коррелированных IV-теноров в 3 независимых фактора. Помимо снижения размерности, PCA автоматически выделяет самые информативные направления вариации. Если все 15 теноров IV двигаются синхронно, `iv_pca_1` захватит ~90% информации, а остальные 14 столбцов — шум.

**Важно:** `shift(1)` применяется после PCA-проекции. Это означает, что PCA-компоненты за сентябрь используются для прогноза на ноябрь (а не на октябрь). Корректная профилактика look-ahead bias.

### 3.6. Волатильность волатильности (vol-of-vol)

```python
iv_std3 = iv.rolling(3).std().shift(1)
features["iv_vol_of_vol_short"] = iv_std3.mean(axis=1)
iv_std6 = iv.rolling(6).std().shift(1)
features["iv_vol_of_vol_long"] = iv_std6.mean(axis=1)
```

**Что делает код:** `rolling(3).std()` — стандартное отклонение IV за 3 месяца для каждого тенора. `.mean(axis=1)` — усреднение по всем 15 тенорам (получаем одно число на месяц). `shift(1)` — сдвиг.

**Какие столбцы создаются (2 штуки):**

| Столбец | Формула | Смысл |
|---|---|---|
| `iv_vol_of_vol_short` | mean(std_3m по всем тенорам IV) | Краткосрочная нестабильность рыночных ожиданий |
| `iv_vol_of_vol_long` | mean(std_6m по всем тенорам IV) | Среднесрочная нестабильность |

**Зачем:** Vol-of-vol (волатильность волатильности) — мета-мера. Высокое значение означает, что рынок не может «определиться»: IV скачет из месяца в месяц. Это отличается от просто высокой IV (стабильно высокий уровень страха vs хаотическое метание). В режиме высокого vol-of-vol ставки ведут себя непредсказуемо; в режиме низкого vol-of-vol — стабильно. Модель может использовать этот сигнал для «калибровки уверенности» — делать более консервативные прогнозы в нестабильные периоды.

## 6. Блок 3. Метрика качества

```python
def weighted_rmse(y_true, y_pred):
    T = y_true.shape[0]
    weights = np.array([WEIGHT_ON] + [WEIGHT_OTHER] * (len(YIELD_TENORS) - 1))
    sq_errors = (y_true - y_pred) ** 2
    weighted_sum = np.sum(weights[np.newaxis, :] * sq_errors)
    rmse = np.sqrt(weighted_sum / T)
    return rmse
```

**Что делает код (пошагово):**

1. `y_true` и `y_pred` — матрицы размером (T, 9), где T = 6 (тестовых месяцев), 9 = теноры.
2. `weights` — вектор [0.4, 0.075, 0.075, 0.075, 0.075, 0.075, 0.075, 0.075, 0.075].
3. `sq_errors = (y_true - y_pred) ** 2` — поэлементный квадрат ошибки, матрица (6, 9).
4. `weights[np.newaxis, :]` — превращает вектор весов из (9,) в (1, 9) для broadcasting.
5. `weights * sq_errors` — каждый столбец ошибки умножается на свой вес, матрица (6, 9).
6. `np.sum(...)` — суммирование всех элементов (по месяцам и тенорам), получаем одно число.
7. Делим на T и извлекаем корень.

**Формула из условия:**

$$\text{RMSE} = \sqrt{\frac{1}{T} \sum_{t} \sum_{\tau} \omega_{\tau} \left(\hat{y}(t, \tau) - y(t, \tau)\right)^2}$$

**Функция `rmse_total`:**

```python
RMSE_total = 0.5 × RMSE^M1 + 0.5 × RMSE^M2
```

Финальная метрика для ранжирования команд. Обе модели вносят одинаковый вклад.

## 7. Блок 4. Модель M1

Словарь `models` хранит 9 независимых Ridge-моделей — по одной на каждый тенор.

### Обучение (`fit`)

```python
def fit(self, yield_df):
    self.feature_engineer = FeatureEngineer(yield_df)
    X = self.feature_engineer.yield_curve_features()
    y = yield_df.copy()

    common_idx = X.dropna().index.intersection(y.dropna().index)
    X = X.loc[common_idx]
    y = y.loc[common_idx]

    self._fit_sklearn(X, y)
```

**Шаги:**

1. Создаёт `FeatureEngineer` и генерирует матрицу признаков X (~79 столбцов).
2. Целевая переменная y — исходная кривая доходности (9 столбцов).
3. `common_idx` — пересечение дат, где нет NaN ни в X, ни в y. Первые ~6 строк теряются из-за лагов и скользящих средних. Из 78 остаётся ~72.
4. Передаёт в `_fit_sklearn`.

### Обучение sklearn-моделей

```python
def _fit_sklearn(self, X, y):
    scaler_X = StandardScaler()
    X_scaled = scaler_X.fit_transform(X)
    self.scalers["X"] = scaler_X

    for tenor in YIELD_TENORS:
        model = Ridge(alpha=1.0)
        model.fit(X_scaled, y[tenor].values)
        self.models[tenor] = model
```

**Шаги:**

1. **Масштабирование:** `StandardScaler` приводит каждый столбец X к среднему 0 и стандартному отклонению 1. Формула: `x_scaled = (x - mean) / std`. Scaler сохраняется — его нужно применить к тестовым данным с теми же параметрами (mean и std обучающей выборки).
2. **Обучение:** Для каждого тенора создаётся Ridge-модель с `alpha=1.0`. Ridge решает задачу: `min ||y - X·β||² + α·||β||²`. Первое слагаемое — точность, второе — регуляризация (штраф за большие коэффициенты). α=1.0 — умеренная регуляризация.
3. Результат: 9 обученных моделей, каждая содержит вектор коэффициентов β длиной ~79.

### Рекурсивный прогноз

```python
def _predict_sklearn(self, yield_df_full, n_steps=6):
    df = yield_df_full.copy()
    predictions = []
    pred_dates = pd.date_range(...)

    for step in range(n_steps):
        fe = FeatureEngineer(df)
        X_new = fe.yield_curve_features()
        X_last = X_new.iloc[[-1]]
        X_scaled = self.scalers["X"].transform(X_last)

        pred_row = {}
        for tenor in YIELD_TENORS:
            pred_row[tenor] = self.models[tenor].predict(X_scaled)[0]
        predictions.append(pred_row)

        new_row = pd.DataFrame(pred_row, index=[pred_dates[step]])
        df = pd.concat([df, new_row])

    return pd.DataFrame(predictions, index=pred_dates, columns=YIELD_TENORS)
```

**Шаги цикла (6 итераций):**

1. **Пересоздаём FeatureEngineer** с текущим DataFrame (включая все предыдущие прогнозы).
2. **Генерируем фичи** для всех строк, берём **последнюю строку** (`iloc[[-1]]`) — это фичи для прогнозируемого месяца.
3. **Масштабируем** с сохранённым scaler (используя mean и std обучающей выборки).
4. **Прогнозируем** 9 ставок — по одной модели на каждый тенор.
5. **Добавляем прогноз** в DataFrame (`pd.concat`) — теперь этот прогноз станет «данными» для следующей итерации.

**Ключевой момент:** На каждой итерации DataFrame растёт на одну строку. Лаги, MA и дельты автоматически подхватывают прогнозированные значения. К шагу 6 (март 2026) модель работает с данными, где последние 5 строк — её собственные прогнозы.

**Почему `TimeSeriesSplit`, а не обычный KFold:**

Обычный KFold перемешивает данные случайно. Модель может обучиться на данных 2024 года и тестироваться на 2020. Это некорректно — в реальности мы не знаем будущего. `TimeSeriesSplit` гарантирует: train всегда хронологически раньше test.

**Структура фолдов (для ~72 строк, 5 фолдов):**

| Фолд | Train | Test |
|---|---|---|
| 1 | Строки 1–12 | Строки 13–24 |
| 2 | Строки 1–24 | Строки 25–36 |
| 3 | Строки 1–36 | Строки 37–48 |
| 4 | Строки 1–48 | Строки 49–60 |
| 5 | Строки 1–60 | Строки 61–72 |

**Важный момент:** `scaler.fit_transform(X_train)` — scaler обучается только на train. `scaler.transform(X_test)` — к test применяются параметры train. Если сделать `fit_transform` на test — утечка информации (scaler «увидит» тестовые данные).
